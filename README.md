**FTN HUD Manager** es un proyecto de vitrina técnica diseñado para explorar la integración de datos en tiempo real entre Counter-Strike 2 y entornos de escritorio avanzados. Este showcase demuestra la capacidad de centralizar el control estético y el análisis de datos en una interfaz robusta desarrollada con tecnologías modernas.

---

## 🌎 Language / Idioma

### **English**
**FTN HUD Manager** is a technical showcase focused on bridging live game data with professional broadcast overlays. This repository acts as a portfolio piece to demonstrate full-stack development skills in competitive gaming environments.

* **Dynamic HUD Control**: Real-time modification of visual elements using WebSocket synchronization.
* **Technical Dashboard**: Data management and tracking for matches and players using SQLite with custom triggers.
* **On-Site Security (HWID Study)**: Implementation of hardware-based authentication as a security study for broadcast environments on external machines.
* **GSI Integration**: Deep dive into Valve's Game State Integration for low-latency data processing.

### **Español**
**FTN HUD Manager** es un showcase técnico centrado en conectar datos de juego en vivo con overlays profesionales. Este repositorio actúa como una pieza de portafolio para demostrar habilidades de desarrollo full-stack en entornos de gaming competitivo.

* **Control Dinámico de HUD**: Modificación en tiempo real de elementos visuales mediante sincronización por WebSockets.
* **Dashboard Técnico**: Gestión y seguimiento de datos para partidas y jugadores usando SQLite con triggers personalizados.
* **Seguridad On-Site (Estudio HWID)**: Implementación de autenticación por hardware como estudio de seguridad para entornos de transmisión en máquinas externas.
* **Integración GSI**: Análisis profundo del Game State Integration de Valve para procesamiento de datos de baja latencia.

---

## 🛠️ Tech Stack / Tecnologías
* **Desktop Layer**: Electron (Custom Main Process handling GSI data).
* **Backend**: Node.js & Express with WebSocket architecture for instant state sync.
* **Frontend**: React.js with Tailwind CSS for high-performance UI components.
* **Database & Auth**: Supabase (Cloud) and SQLite (Local) with automated SQL triggers.
